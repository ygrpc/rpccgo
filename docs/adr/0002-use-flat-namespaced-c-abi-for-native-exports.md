# Use a flat namespaced C ABI for native exports

rpccgo uses flat C ABI boundaries for native exports: cross-runtime calls must not pass `struct` or `struct*`, and C native server callbacks are registered through flat callback parameters instead of a callback table struct. Registration accepts method-level partial implementations: a missing unary callback means that method is unimplemented, while streaming method callbacks are validated atomically and must be either all missing or all present. A registration with every method missing is still a registered all-unimplemented server. Per-method C registration accumulates callbacks only when the current registered server has the same `rpcruntime.ServerKind`; otherwise it creates a new cgo adapter and replaces the current registered server. Exported C symbols use `rpccgo_<contract>_<namespace>_<Service>_<Method>[_<operation>]`, where `namespace` defaults to the Go package name and can be explicitly overridden on collision; this keeps symbols short while avoiding the old collision-prone `rpccgo_native_go_<Service>_<Method>` shape and avoids embedding endpoint-language prefixes that add no information once operation names distinguish call and registration APIs.
